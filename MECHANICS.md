## What This Is

StenoAI (repo/codename; the user-facing brand is now **"Steno"** after the #227 copy rebrand) is a privacy-first desktop app (macOS signed/notarised; Windows alpha) that records meetings, transcribes them locally, and summarises them with an LLM. It is a **thin Electron shell over a PyInstaller-bundled Python CLI** — there is no long-running backend service. Every operation (transcribe, summarise, config read/write) is a one-shot `child_process.spawn` of the bundled `stenoai` binary. The only long-lived children are the Parakeet **`transcribe-stream`** live-partials sidecar (during a recording) and the bundled **`ollama serve`** — the legacy mic `record` subprocess is retired (see Recording, below). Since #276 the pipeline is **transcript-first**: transcription always produces a saved transcript, and summary ("notes") generation is a *decoupled, optional* second stage gated by an `auto_summarize_enabled` toggle — when off, the app saves a transcript-only note the user later fills in via a floating **"Generate notes"** button (#276, `luffy/transcribe-pill-dock`). An optional **enterprise dimension** routes AI + note backup through a signed-in org adapter so the desktop never holds provider keys. User data lives in `~/Library/Application Support/stenoai/` (macOS) / `%APPDATA%/stenoai/` (Windows).

## Core Data Flow: Record → Transcribe → (optionally) Generate Notes → Save

Since #276 this is **transcript-first**: transcription and note-generation are two decoupled stages, and the second is gated by `auto_summarize_enabled` (default `True`).

1. **Start** — renderer calls `ipc().recording.start(name)`; `start-recording-ui` takes a **single renderer-driven path** for both mic and system audio (no mic-XOR-system fork). The renderer's `useSystemAudioCapture` hook captures the mic (+ system loopback when the Settings toggle is on) via Web Audio/MediaRecorder and streams its WebM incrementally to disk; main just flips `systemAudioRecordingActive = true`, records `recordingRuntimeState`, and — **only if the engine is Parakeet** — spawns the `transcribe-stream` live-partials sidecar (`spawnLiveTranscribe`). Whisper recordings skip the sidecar and use the post-stop pipeline.
2. **Live transcription (Parakeet only)** — the sidecar runs `_LiveVadPipeline`: Silero VAD (`src/silero_vad.py`) over 16 kHz mono; each finished utterance is batch-transcribed by Parakeet and emitted as `LIVE_SEG:<json>` (now with true per-channel `[You]`/`[Others]` attribution, #284, and `[MM:SS]` timestamps, #272); trailing audio is re-transcribed for partials. Whisper has no live path (`SUPPORTS_PARTIALS=False`).
3. **Stop → queue** — `stop-recording-ui` clears `systemAudioRecordingActive`, stops the live sidecar, and the renderer hands the finished WebM to `process-system-audio-recording`, which enqueues it. (Local-file **import** — picker or drag-drop via `useImportAudio.ts`, #185 — feeds the same queue.) `processNextInQueue()` spawns `process-streaming`, guarded by an inactivity watchdog reset on `HEARTBEAT:` lines (frozen across system sleep) and mirrored to the redacted on-disk processing log (#224/#293).
4. **Transcribe (always)** — `WhisperTranscriber` (`src/transcriber.py`, name is legacy) dispatches to Parakeet via `src/parakeet.py` (or whisper.cpp on crash fallback). ffmpeg pre-processes (high-pass 90 Hz + loudnorm). Stereo system-audio is diarised into `[You]`/`[Others]`. Long files chunk at `PARAKEET_CHUNK_DURATION_S=60s` with 15 s overlap. `MeetingPipeline.transcribe_audio()` (`simple_recorder.py:320`) **unconditionally** writes `<stem>_transcript.txt` (`:381`) — the transcript is the primary artifact.
5. **Note-generation gate** — `process-streaming` (`simple_recorder.py:730`) checks `get_auto_summarize_enabled()` (`src/config.py:897`). **If off**, it writes a transcript-only summary file (`notes_generated: False` frontmatter, `simple_recorder.py:541/893`) and returns `SUMMARY_SKIPPED` — no LLM call. **If on**, it proceeds to summarise.
6. **Summarise (conditional)** — `OllamaSummarizer.summarize_transcript_streaming` builds `_create_markdown_prompt` and streams via the active `ai_provider` (local Ollama / remote / cloud / adapter). Chunks are base64'd and printed as `CHUNK:` lines, terminated by an exact-match `STREAM_COMPLETE` sentinel; main forwards them to the renderer. The backend forces LF-only stdout on Windows (`reconfigure(newline="\n")`, `simple_recorder.py:38`) so these exact-match sentinels never carry a stray `\r` and strand the UI "in analysis".
7. **Deferred notes ("Generate notes" button)** — a transcript-only note shows a floating **`GenerateNotesBar`** in the renderer. Clicking it calls `ipc().meetings.reprocess(summaryFile, regenTitle=false, name)` → the `reprocess-meeting` handler (`main.js:1883`) → CLI `reprocess` (`simple_recorder.py:2350`), which re-summarises **from the existing transcript** (no re-transcription) and streams `CHUNK:` back. The button lives at the App level, not in the detail view, so a **zustand bridge** (`hooks/reprocessBridgeStore.ts`) relays the detail's `startReprocess` callback + streaming state up to it (see Component Reference).
8. **Save** — `_parse_streamed_markdown` turns the stream into structured JSON; `_atomic_write_json` (tempfile + `os.replace`) writes `<stem>_summary.json`/`.md` to `output/`. A title is generated separately when the name is auto/placeholder. If signed in with auto-backup, main shares it to the org adapter (presign → S3 PUT → register).

## Component Reference

**Electron main (`app/main.js`)** — *Owns*: window/tray, 140 IPC handlers, backend spawning, recording globals (`systemAudioRecordingActive` + `recordingRuntimeState` + `currentRecordingSessionName`; `processingQueue`), org session + AI-provider lock. Note `currentRecordingProcess` is now **vestigial** — always `null` since the Python mic subprocess was retired; `systemAudioRecordingActive` is the real "is-recording" flag (its name predates the unification and now covers all recordings). *Interface*: `ipcMain.handle(name, …)` ↔ preload. *Invariants*: at most one active recording at a time; provider-state mutations serialized on the provider queue; `STENOAI_USER_DATA_DIR` honored so tests never touch real data.

**Preload bridge (`app/preload.js`)** — *Owns*: the only channel renderer→main. *Interface*: `window.stenoai.<group>.<call>` wrapping `ipcRenderer.invoke/send/on`; `subscribeQueryStream` multiplexes streaming chunks by id. *Invariant*: renderer never touches `ipcRenderer` directly.

**Renderer IPC (`lib/ipc.ts`, ~1.0k lines)** — *Owns*: the typed bridge surface + `Result<T>`/`unwrap` envelope. *Invariant*: it's the hand-maintained source of truth — a new handler must be added to `main.js` + `preload.js` + `ipc.ts` together (no compile-time link between them; this is the #1 fragile seam). #288 added an IPC-contract test as a partial backstop.

**Note-generation bridge (`hooks/reprocessBridgeStore.ts` + `components/GenerateNotesBar.tsx`)** — *Owns*: relaying a transcript-only note's "generate notes" action from `MeetingDetail` (which owns the reprocess mutation) up to the App-level floating bar. *Interface*: a zustand store with `{summaryFile, streaming, start}` + `publish()`/`clear(summaryFile)`; the bar renders only when `summaryFile === activeSummaryFile && start !== null && !transcriptOpen`, disabled while `streaming`. *Invariants*: `clear()` no-ops unless the file still owns the bridge (guards a late unmount from wiping a newer detail's state); `GenerateNotesBar` and `TranscriptBar` share the **bottomOffset=72** dock band above the Ask bar and must stay mutually exclusive (the bar hides when `transcriptOpen`).

**Settings (`routes/Settings.tsx` + `routes/settings/`)** — *Owns*: the settings UI, split (#289) into a ~150-line tab-shell + per-tab components: `GeneralTab`, `TranscriptionTab`, `AiTab`, `TemplatesTab`, `OrganisationTab`, `AdvancedTab`, `DeveloperTab`, with shared `primitives.tsx` / `model-card.tsx` / `languages.ts`. Several tabs now carry colocated `*.test.tsx`.

**Org hooks (`hooks/useOrg.ts`)** — *Owns*: org session, policy gate, share/unshare, backup-state. *Interface*: `useOrgSession` (schedules a precise JWT-expiry invalidation), `useOrgPolicy` (keyed per `orgId`), `useSharedNotesGate`. *Invariant*: policy cache is per-org so a previous org can't leak an enabled gate; gate fails **open** on policy error (server enforces the real boundary).

**Summarizer (`src/summarizer.py`)** — *Owns*: all four provider paths and prompt construction. *Interface*: `summarize_transcript[_streaming]`, `_create_markdown_prompt` (streaming) / `_create_permissive_prompt` (JSON). *Invariants*: adapter sends only `{messages, max_tokens}` (no model/system) so the adapter picks the model; Ollama touched via HTTP only (never the binary except `serve`).

**Config (`src/config.py`)** — *Owns*: `config.json`, data-dir resolution, model registry, template CRUD, migrations. *Interface*: `get_config()` singleton, ~50 getters/setters (incl. `get/set_auto_summarize_enabled`, template getters); `get_adapter_url/token` read from env (set by Electron). *Invariants*: `STENOAI_USER_DATA_DIR` > custom `storage_path` > platform default; atomic writes; corrupt files quarantined to `.corrupt`. **Cross-process saves are `filelock`-guarded** (#291, `config.json.lock`): under the lock `_save()` re-reads the on-disk config as the merge base and overlays only the top-level keys this process changed (diffed against a load-time snapshot, nested dicts merged recursively), so two concurrent CLI processes editing disjoint keys never lose each other's writes. On lock timeout it degrades to a plain atomic write rather than stalling the app.

**Templates (`src/templates.py` + `TemplatesTab.tsx`)** — *Owns*: the note/report template gallery. *Interface*: `BUILTIN_TEMPLATES` (a locked `standard` structured template that drives the JSON summary, plus editable markdown templates like `product-demo`/`sales-call`/`one-on-one`/`standup`); user edits live in `config.json` as `custom_templates` (new) + `template_overrides` (edits to built-ins that survive upgrades) + `default_template_id`. *Interface (CLI)*: `generate-report <summary_file> <template_id>` renders a per-meeting report sidecar via the LLM (`simple_recorder.py:~2595`). *Invariant*: the `standard` template is `locked` and non-editable.

**Transcription (`src/parakeet.py` + backends)** — *Owns*: ASR dispatch. *Interface*: `transcribe_file/_samples`, `SUPPORTS_PARTIALS`, `PARAKEET_CHUNK_DURATION_S`. *Invariant*: call sites never branch on engine; platform chooses MLX (mac GPU) vs onnx-asr (CPU). 16 kHz mono only.

**Org adapter (`../stenoai-enterprise`, separate repo)** — FastAPI: `/auth/login` (hardcoded `users.json`, demo), `/policy` (env-flag policy), `/meetings` (S3-backed, enforces `shared_notes_enabled` server-side), `/ai/chat[/stream]` (Anthropic proxy, default `claude-sonnet-4-6`).

## Design Decisions

1. **No daemon; every op is a subprocess.** Simpler lifecycle, crash-isolated, but state must round-trip through JSON files and each invocation re-pays Python/import startup.
2. **Recording is fully renderer-driven; Python is only a transcription worker** (#223–226). Both mic and system audio are now captured renderer-side (Web Audio/MediaRecorder → incremental WebM on disk); Python is spawned only for the Parakeet live-partials sidecar and the post-stop `process-streaming` pipeline. The old signal-controlled Python `record` subprocess was retired chiefly because signals don't work on Windows. *Consequence*: one uniform capture path + uniform sleep/auto-pause handling (the renderer pauses its `MediaRecorder`), and live recording state no longer needs a JSON file — but the recording state machine now spans renderer ↔ main, and the Electron-side flags (`systemAudioRecordingActive`) keep a legacy name.
3. **Provider routed by `ai_provider` + an Electron-side hard lock.** Sign-in forces `adapter`; sign-out restores the remembered `.pre-adapter-provider`. Keeps keys off the desktop, at the cost of intricate reconcile/restore logic (`reconcileAiProviderWithOrgSession`).
4. **JWT expiry handled with a precise `setTimeout`, not polling** (`useOrg.ts:45`). No stale "signed in" UI; clamps to ~23 days for very long TTLs.
5. **Policy gate fails open on the client, enforced on the server.** UI resilience without a security risk — `/meetings` collapses to owner-only server-side regardless of the client.
6. **Adapter request omits model + system.** The org controls the model centrally; downside is summary phrasing differs from a user-picked cloud model. The phrasing is pinned in the *prompt text itself* (not a post-process): both builders — `_create_markdown_prompt` (`summarizer.py:977`) and `_create_permissive_prompt` (`:676`) — carry an explicit "write the overview directly, do not refer to 'the transcript' / open with 'The transcript discusses…'" instruction. Guarded by the deterministic `summarize-contract.t2` e2e (now on `main`).
7. **Long-meeting chunking at a hard 60 s.** Parakeet-mlx would OOM batching a whole long file; 60 s/15 s-overlap windows bound memory. Overlap-dedup epsilon can rarely miss true repeats.
8. **Tiered e2e (T1 hermetic / T2 real backend / T3 nightly), keystone = `STENOAI_USER_DATA_DIR`.** Fast per-PR signal without models; real-wiring cross-check; model-bearing specs split to `@pipeline`. T1 mock IPC can drift from real wire shape — T2 is the truth.
9. **Atomic JSON everywhere** (tempfile + `os.replace`). Safe against concurrent CLI processes; needs retry on Windows `PermissionError`.
10. **Streaming protocol is exact-match line sentinels (`STREAM_COMPLETE`, `CHAT_STREAM_COMPLETE`) over stdout.** Cheap and language-agnostic, but exact equality is fragile to newline translation: Windows text-mode stdout turns `\n`→`\r\n`, so `line === 'STREAM_COMPLETE'` silently failed and the summary UI hung "in analysis" (#222, prior class #220). Fixed at the source — `reconfigure(newline="\n")` on win32 — with the JS handlers also splitting on `/\r?\n/` as defense-in-depth.
11. **Transcript-first pipeline; notes are decoupled and optional** (#276). Transcription always saves `<stem>_transcript.txt`; summary generation is a second stage gated by `auto_summarize_enabled`. *Why*: the transcript has value on its own, LLM summarisation is the slow/expensive/failure-prone step, and users increasingly want the raw transcript without waiting on (or paying for) an LLM. *Consequence*: a note can exist in a "transcript-only" state (`notes_generated: False`), so the UI needs an explicit **Generate notes** affordance and the summary parser/backup path must tolerate a note with no summary body. Adds an App↔detail zustand bridge to hoist the action to a floating bar.
12. **Cross-process config writes are `filelock`-guarded with a re-read/merge** (#291), not last-writer-wins. *Why*: two CLI subprocesses (e.g. a settings write racing a pipeline's title write) could clobber each other's disjoint keys. *Consequence*: correctness under concurrency at the cost of a lock file + a merge-base re-read; degrades to a plain atomic write on lock timeout so a stuck lock never freezes the app.
13. **Debug-log privacy by source reduction, not post-hoc scrubbing** (#293–294). `diagnostics-filter.js` redacts sensitive CLI args (`ARGS_ECHO_REDACTORS`) and forwards only an **allowlist** of content-free protocol stdout lines — meeting content (CHUNK/TITLE/LIVE_SEG/query answers) never enters the shareable log in the first place. *Consequence*: the debug export is safe to share, but any new content-bearing stdout marker must be explicitly considered against the allowlist or it's silently dropped from diagnostics.
14. **`setup-check` emits structured JSON, not scraped emoji text** (#290). The CLI prints `{allGood, checks:[{name, ok, status, detail}]}`; `setup-check-parse.js` validates the schema + invariants. *Why*: the renderer previously parsed emoji prefixes (`❌`/`⚠️`) out of human text — brittle and locale-fragile.

## Hotspots & Risks

- **`lib/ipc.ts` ↔ `preload.js` ↔ `main.js` triple** — no compile-time link; a drift ships a runtime "undefined is not a function". Highest-traffic seam in the app. #288's IPC-contract test catches missing declarations but not signature/shape drift.
- **`GenerateNotesBar` / `TranscriptBar` shared `bottomOffset=72` dock band** — the two floating bars share one slot above the Ask bar and rely on being mutually exclusive (bar hides when `transcriptOpen`). If a future gate lets both be visible at once they collide; the invariant is enforced only by convention + a T1 spec (`generate-notes-bar.t1`), not the layout. The `reprocessBridgeStore.clear()` ownership check is the guard against a late-unmount race wiping a newer detail's state.
- **AI-provider lock** (`main.js:5862–5990`) — single serialized promise queue + 30 s memo + 10 min keychain-lock grace. A failed Python `set-ai-provider` isn't retried; `org-status` reconcile is fire-and-forget, so a sign-in-then-immediate-AI-op can race the switch.
- **Org session decrypt** (`main.js:7879`) — `safeStorage.decryptString` throws on a locked keychain (post-wake); three-state load + hardcoded grace window; no retry. On Windows e2e it may legitimately skip (DPAPI on a headless runner).
- **Recording stop vs sidecar shutdown** (`main.js:3211`) — `systemAudioRecordingActive` cleared before confirming the live `transcribe-stream` sidecar is dead; a slow sidecar can report `hasRecording=false` while still emitting.
- **Inactivity watchdog across sleep** (`main.js:2832`) — a job created in the suspend→sleep millisecond window can get an unarmed timer the thaw loop never revisits.
- **`onnx-asr` metadata + lazy MLX imports** (`stenoai.spec:126–176`) — `copy_metadata`/`collect_submodules` are sledgehammers; a new upstream submodule fails silently at runtime until re-collected. Off-darwin only for the metadata.
- **CI exec-bit strip (macOS)** (`e2e.yml`) — `upload-artifact` drops the exec bit; jobs must `chmod +x` or every spawn fails silently.
- **Playwright `--grep` exits 0 on zero matches** — a dropped `@pipeline`/`@long-meeting` tag silently runs zero tests; guards assert selection.
- **Hand-rolled stdout line parsers** (`main.js`: `reprocess`, `query-transcript-stream`, `processNextInQueue` `~2815`, the `transcribe-stream` LIVE_* handler `~2513`, plus the chat handler) — each splits/buffers stdout independently and two patterns coexist (exact-match `=== 'STREAM_COMPLETE'` vs `startsWith` markers). #222 made them CRLF-tolerant, but a tracked follow-up is to unify them behind one buffered line reader (fixes a rare cross-chunk-split case where a sentinel straddles two `data` events).
- **`simple_recorder.py` god-file** (~4.5k lines, ~87 commands) and **`main.js`** (~9.5k lines) — both are central, high-blast-radius, and mix many concerns; both still growing. The renderer's `Settings.tsx` monolith was tamed (#289) but these two backends were not.
- **Known-broken baselines** (non-blocking): `lint:renderer` (eslint v9 flat config), ruff baseline in `simple_recorder.py`.
