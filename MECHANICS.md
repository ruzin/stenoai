_Last updated: 2026-06-21. Auto-generated by /usopp._

## What This Is

Steno (product name; repo, binary, and data dir are still `stenoai`) is a privacy-first desktop app (macOS signed/notarised; Windows alpha) that records meetings, transcribes them locally, and summarises them with an LLM. It is a **thin Electron shell over a PyInstaller-bundled Python CLI** — there is no long-running backend service. Every operation (transcribe, summarise, config read/write) is a one-shot `child_process.spawn` of the bundled `stenoai` binary, except a long-lived live-transcription sidecar (`transcribe-stream`) and the bundled `ollama serve`. **Audio capture itself is no longer a Python subprocess** — as of #225/#226 it is fully renderer-driven (MediaRecorder/Web Audio) on every platform, streamed to disk as WebM. An optional **enterprise dimension** routes AI + note backup through a signed-in org adapter so the desktop never holds provider keys. User data lives in `~/Library/Application Support/stenoai/` (macOS) / `%APPDATA%/stenoai/` (Windows).

## Tech Stack

- **Electron main** (`app/main.js`, ~8.5k lines) — Node; owns window/tray/deep-links, audio-capture orchestration, and orchestrates everything via `ipcMain.handle` (~141 handlers). Spawns the Python backend.
- **Renderer** (`app/renderer/src`, ~18.5k LOC) — React + TypeScript + Vite, `contextIsolation: true`. TanStack Query for server-state, zustand for UI-state, hash-based routing (no router lib). **Owns audio capture** via MediaRecorder (`useSystemAudioCapture`). Also owns: local audio **import** (picker + drag-and-drop `ImportDropZone`, #185, routed through the same processing queue), the global **⌘K command palette** note search (`CommandPalette` + shared `noteSearch` matcher, #213/#231), and an **optimistic delete** that drops the row from the query cache instead of re-listing (#189/#204). Renderer hook/logic now has **vitest** unit tests (jsdom) alongside the Playwright tiers.
- **Python CLI** (`simple_recorder.py`, ~3.6k lines, ~75 click commands) + `src/` modules — transcription/summarisation pipeline (no longer captures audio).
- **Audio/ML**: renderer MediaRecorder (capture; `sounddevice` retained for device enumeration/VAD on the live sidecar), Parakeet TDT v3 (`parakeet-mlx` on macOS / `onnx-asr` on Win/Linux) with whisper.cpp fallback, Silero VAD (live partials), bundled Ollama or cloud/org-adapter LLM.
- **Packaging**: PyInstaller (`stenoai.spec`) → `dist/stenoai/`; electron-builder re-bundles it as `extraResources`. Bundled `ollama` + `ffmpeg` in `bin/`.

## Core Data Flow: Record → Transcribe → Summarise → Save

1. **Start** — renderer calls `ipc().recording.start(name)`; `start-recording-ui` (`main.js:2867`) is now uniform across platforms: it flips `systemAudioRecordingActive=true`, resets the live-transcript buffer, and (Parakeet only) spawns the `transcribe-stream` sidecar (`spawnLiveTranscribe`). The renderer's `useSystemAudioCapture` hook sees `hasRecording=true` on the next poll and starts MediaRecorder, capturing the mic (+ system loopback when the Settings toggle is on). **There is no longer a mic-XOR-system fork** — the legacy signal-controlled Python `record` subprocess (unworkable on Windows) is retired.
2. **Capture → disk** — the renderer streams WebM chunks to main via `append-system-audio-chunk` (`main.js:5806`), which appends them to a file under `recordings/`; `close-system-audio-file` (`5837`) finalises it. The live audio is also tapped to the sidecar via `live-transcribe-chunk` (`main.js:2229`).
3. **Live transcription (Parakeet only)** — the `transcribe-stream` sidecar runs `_LiveVadPipeline` (`simple_recorder.py:1257`) with Silero VAD (`src/silero_vad.py`) over 16 kHz mono; each finished utterance is batch-transcribed by Parakeet and emitted as `LIVE_SEG:<json>`; trailing audio is re-transcribed for partials. Whisper has no live path (`SUPPORTS_PARTIALS=False`).
4. **Stop → queue** — `stop-recording-ui` (`main.js:3048`) clears `systemAudioRecordingActive`, closes the sidecar (`stopLiveTranscribe`), and the renderer's stopCapture flow finalises the WebM and hands it to `process-system-audio-recording` (`main.js:5893`), which enqueues it. `processNextInQueue()` (`main.js:2670`) spawns `process-streaming`, guarded by an inactivity watchdog reset on `HEARTBEAT:` lines (frozen across system sleep).
5. **Transcribe** — `WhisperTranscriber` (`src/transcriber.py`, name is legacy) dispatches to Parakeet via `src/parakeet.py` (or whisper.cpp on crash fallback). ffmpeg pre-processes (high-pass 90 Hz + loudnorm) and decodes the WebM. Stereo system-audio is diarised into `[You]`/`[Others]`. Long files chunk at `PARAKEET_CHUNK_DURATION_S=60s` with 15 s overlap.
6. **Summarise** — `OllamaSummarizer.summarize_transcript_streaming` (`src/summarizer.py:1056`) builds `_create_markdown_prompt` (`:1001`) and streams via the active `ai_provider` (local Ollama / remote / cloud / adapter). Chunks are base64'd and printed as `CHUNK:` lines, terminated by an exact-match `STREAM_COMPLETE` sentinel; main forwards them to the renderer. The backend forces LF-only stdout on Windows (`reconfigure(newline="\n")`, `simple_recorder.py:38`) so these exact-match sentinels never carry a stray `\r` and strand the UI "in analysis".
7. **Save** — `_parse_streamed_markdown` turns the stream into structured JSON; `_atomic_write_json` (tempfile + `os.replace`) writes `<stem>_summary.json` to `output/`. A title is generated separately when the name is auto/placeholder. If signed in with auto-backup, main shares it to the org adapter (presign → S3 PUT → register).

**Preload bridge (`app/preload.js`)** — *Owns*: the only channel renderer→main. *Interface*: `window.stenoai.<group>.<call>` wrapping `ipcRenderer.invoke/send/on`; `subscribeQueryStream` multiplexes streaming chunks by id. *Invariant*: renderer never touches `ipcRenderer` directly.

**Renderer IPC (`lib/ipc.ts`, 877 lines)** — *Owns*: the typed bridge surface + `Result<T>`/`unwrap` envelope. *Invariant*: it's the hand-maintained source of truth — a new handler must be added to `main.js` + `preload.js` + `ipc.ts` together (no compile-time link between them; this is the #1 fragile seam).

**Org hooks (`hooks/useOrg.ts`)** — *Owns*: org session, policy gate, share/unshare, backup-state. *Interface*: `useOrgSession` (schedules a precise JWT-expiry invalidation), `useOrgPolicy` (keyed per `orgId`), `useSharedNotesGate`. *Invariant*: policy cache is per-org so a previous org can't leak an enabled gate; gate fails **open** on policy error (server enforces the real boundary).

**Summarizer (`src/summarizer.py`)** — *Owns*: all four provider paths and prompt construction. *Interface*: `summarize_transcript_streaming` (`:1056`), `_create_markdown_prompt` (`:1001`, streaming) / `_create_permissive_prompt` (`:693`, JSON). *Invariants*: adapter sends only `{messages, max_tokens}` (no model/system) so the adapter picks the model; Ollama touched via HTTP only (never the binary except `serve`).

**Config (`src/config.py`)** — *Owns*: `config.json`, data-dir resolution, model registry, migrations. *Interface*: `get_config()` singleton, ~50 getters/setters; `get_adapter_url/token` read from env (set by Electron). *Invariant*: `STENOAI_USER_DATA_DIR` > custom `storage_path` > platform default; atomic writes; corrupt files quarantined to `.corrupt`.

## Design Decisions

1. **No daemon; every op is a subprocess.** Simpler lifecycle, crash-isolated, but state must round-trip through JSON files and each invocation re-pays Python/import startup.
2. **All audio capture is renderer-driven; Python is a pure worker (#223/#225/#226).** Both mic and system audio are captured by a single renderer MediaRecorder and streamed to disk as WebM; Python is spawned only for the live `transcribe-stream` sidecar and post-stop `process-streaming`. This unifies the path across macOS/Windows (the old signal-controlled Python `record` subprocess was unworkable on Windows) and removes the mic-XOR-system fork, at the cost of moving capture reliability into the renderer (a torn-down renderer / errored MediaRecorder must be reconciled by main on stop).
3. **Provider routed by `ai_provider` + an Electron-side hard lock.** Sign-in forces `adapter`; sign-out restores the remembered `.pre-adapter-provider`. Keeps keys off the desktop, at the cost of intricate reconcile/restore logic (`reconcileAiProviderWithOrgSession`).
4. **JWT expiry handled with a precise `setTimeout`, not polling** (`useOrg.ts:45`). No stale "signed in" UI; clamps to ~23 days for very long TTLs.
5. **Policy gate fails open on the client, enforced on the server.** UI resilience without a security risk — `/meetings` collapses to owner-only server-side regardless of the client.
6. **Adapter request omits model + system.** The org controls the model centrally; downside is summary phrasing differs from a user-picked cloud model. The phrasing is pinned in the *prompt text itself* (not a post-process): both builders — `_create_markdown_prompt` (`summarizer.py:1001`) and `_create_permissive_prompt` (`:693`) — carry an explicit "write the overview directly, do not refer to 'the transcript' / open with 'The transcript discusses…'" instruction. Guarded by the deterministic `summarize-contract.t2` e2e (now on `main`).
7. **Long-meeting chunking at a hard 60 s.** Parakeet-mlx would OOM batching a whole long file; 60 s/15 s-overlap windows bound memory. Overlap-dedup epsilon can rarely miss true repeats.
8. **Tiered e2e (T1 hermetic / T2 real backend / T3 nightly), keystone = `STENOAI_USER_DATA_DIR`.** Fast per-PR signal without models; real-wiring cross-check; model-bearing specs split to `@pipeline`. T1 mock IPC can drift from real wire shape — T2 is the truth. Below the e2e tiers sits a **unit layer** (#204): `node:test` for the main-process `processing-log` module + **vitest** (jsdom) for renderer hooks, both run by the `Unit (node:test + vitest)` job in `e2e.yml` via `npm run test:unit`.
9. **Atomic JSON everywhere** (tempfile + `os.replace`). Safe against concurrent CLI processes; needs retry on Windows `PermissionError`.
10. **Pre-meeting reminders are scheduled in the Electron main process, not Python (#230).** A background calendar poll arms per-event `setTimeout`s that fire a native heads-up notification before a meeting starts; the `show-premeeting-notification` IPC is a test seam so e2e never depends on wall-clock timers. Keeps the reminder logic next to the notification + calendar-token machinery already in main, at the cost of being session-bound in-memory state (timers don't survive a restart).
11. **Streaming protocol is exact-match line sentinels (`STREAM_COMPLETE`, `CHAT_STREAM_COMPLETE`) over stdout.** Cheap and language-agnostic, but exact equality is fragile to newline translation: Windows text-mode stdout turns `\n`→`\r\n`, so `line === 'STREAM_COMPLETE'` silently failed and the summary UI hung "in analysis" (#222, prior class #220). Fixed at the source — `reconfigure(newline="\n")` on win32 — with the JS handlers also splitting on `/\r?\n/` as defense-in-depth.

## Hotspots & Risks

- **`lib/ipc.ts` ↔ `preload.js` ↔ `main.js` triple** — no compile-time link; a drift ships a runtime "undefined is not a function". Highest-traffic seam in the app.
- **AI-provider lock** (`providerStateQueue` `main.js:5356`; `reconcileAiProviderWithOrgSession` `5387`) — single serialized promise queue + 30 s memo + keychain-lock grace. A failed Python `set-ai-provider` isn't retried; `org-status` reconcile is fire-and-forget, so a sign-in-then-immediate-AI-op can race the switch.
- **Org session decrypt** (`main.js:7879`) — `safeStorage.decryptString` throws on a locked keychain (post-wake); three-state load + hardcoded grace window; no retry. On Windows e2e it may legitimately skip (DPAPI on a headless runner).
- **Recording stop vs sidecar shutdown** (`main.js:3048`) — `systemAudioRecordingActive` cleared before confirming the sidecar is dead; a slow sidecar can report `hasRecording=false` while still emitting. With capture now renderer-driven, a torn-down renderer that never reports `system-audio-recording-state=false` is reconciled only by this defensive clear.
- **Pre-meeting scheduler** (`main.js:7340–7400`) — in-memory `premeetingTimers` (eventId→setTimeout) + `premeetingFiredIds` set, both reset per app session; a long-running app with a flaky calendar poll could double-fire or miss a banner around reschedule (`premeetingRescheduleTimer`). E2E drives the `show-premeeting-notification` IPC seam directly rather than the timer.
- **Inactivity watchdog across sleep** (`main.js:2832`) — a job created in the suspend→sleep millisecond window can get an unarmed timer the thaw loop never revisits.
- **`onnx-asr` metadata + lazy MLX imports** (`stenoai.spec:126–176`) — `copy_metadata`/`collect_submodules` are sledgehammers; a new upstream submodule fails silently at runtime until re-collected. Off-darwin only for the metadata.
- **CI exec-bit strip (macOS)** (`e2e.yml`) — `upload-artifact` drops the exec bit; jobs must `chmod +x` or every spawn fails silently.
- **Playwright `--grep` exits 0 on zero matches** — a dropped `@pipeline`/`@long-meeting` tag silently runs zero tests; guards assert selection.
- **Five hand-rolled stdout line parsers** (`main.js` `reprocess` `1581`, `query-transcript-stream` `1785`, `processNextInQueue` `2913`, `start-recording-ui` `3195`, plus the chat handler) — each splits/buffers stdout independently and two patterns coexist (exact-match `=== 'STREAM_COMPLETE'` vs `startsWith` markers). #222 made them CRLF-tolerant, but a tracked follow-up is to unify them behind one buffered line reader (fixes a rare cross-chunk-split case where a sentinel straddles two `data` events).
- **`simple_recorder.py` god-file** (~3.6k lines, ~75 commands) and **`main.js`** (~8.5k lines) — both are central, high-blast-radius, and mix many concerns. (`simple_recorder.py` shrank with the retired Python `record` path.)
- **Dev/prod data-dir split** (`resolveRecordingsDir` `main.js`; `get_data_dirs` `src/config.py`) — dev JS (`!app.isPackaged`) uses repo-relative `recordings/`+`output/`, but the dev app spawns the *frozen* bundle (`dist/stenoai`) which resolves dirs as **prod** (`~/Library/...`). So under `npm start` the JS and its backend disagree; e.g. the #185/#204 import-collision dedup checks the empty repo `output/` and overwrites the real note. Packaged builds agree (both `~/Library`), so it's dev-only — tracked in #233 (and its e2e blind spot: collision specs seed the note in the JS dir, never driving the full Python round-trip).
- **Known-broken baselines** (non-blocking): `lint:renderer` (eslint v9 flat config), ruff baseline in `simple_recorder.py`.
